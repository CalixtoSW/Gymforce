## O que foi feito

Implementado sistema de autenticação completo com JWT (access + refresh), registro/login/refresh/me, middleware de autenticação com `HTTPBearer`, autorização por roles, CRUD de usuários (admin/recepção), model `User` com enums e migration inicial da tabela `users`.

### Decisões tomadas durante a execução

1. Testes rodam sem Docker usando SQLite assíncrono em memória (`aiosqlite`) com override de `get_db`.
2. Para manter o hash bcrypt funcional com `passlib`, foi fixado `bcrypt<5` no `pyproject`.
3. Regras de role foram aplicadas via dependency factory `require_role(*roles)` para reutilização em endpoints protegidos.
