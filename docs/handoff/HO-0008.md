### Decisões tomadas durante a execução

1. O service de gamificação trata falhas de Redis de forma resiliente para não quebrar endpoint de negócio quando Redis estiver indisponível.
2. A atualização de streak inicializa valores default defensivamente para evitar `None` em ambiente de teste/inserção sem flush prévio.
